L20 — Robustness

Topic: Robustness

Idea: Want learned models to be robust to small changes in the input

This is a problem of smoothness

e.g. for points that differ by a small amount, function has the same prediction.

Model Errors ⇒ Poor Robustness

Sources of error:

  1. Model Misspecified: wrong class, wrong parameters, etc.
  1. Measurement Error: when getting dataset
  1. Dirty Data: Dataset does not reflect the generating distribution: inconsistent, mislabeled, or incomplete.

These problems can be exploited by the Adversary

Adversary

  • Purposefully tries to cause errors
  • Takes advantage of 3 ways for model to not be robust.
  • Does so by creating adversarial example: adds a perturbation (small change to the existing value).

Norms

  • measures size of the changes (how similar the true fff﻿ would judge the inputs)
  • choice of norm determines type and influence of the attack
    • eg. change 1 image pixel or all pixels slightly

Finding the Pertubation

  • mimicks how the model itself is trained
  • optimize inputs to maximize loss ⇒ gradient ascent
  • We will choose our distance metric carefully to make the constraint
    satisfaction easy.
  • Fast Gradient Descent
    • Because we are using L∞ norm, can change all elements by ε.
    • P(x) projects x back into the ε-box centred at x
    • simple, and doesn’t permit the gradient to control the size of the steps taken along any axis.
  • Projected Gradient Descent
    • Don’t threshold the gradient, allowing for smaller steps.
    • more powerful algorithm that leads to better attacks, i.e., attacks requiring a smaller perturbation budget

Diff types of attacks

  • Untargeted: any missclassification, as long as it’s wrong
    • (everything above)
  • Targeted: specific class to be chosen; by modifying loss, penalizes model for not choosing right class
    • Augment the loss function with a term that drives classification towards the desired label, ccc﻿:

Defence

  1. adversarial training (train with adversarial examples; perturbations).
  1. choose a better architecture to begin with — incorporate expert knowledge of what’s important (ie. not small variations that don’t matter).
  1. Learn to flag examples as adversarial, so they aren’t used in the model.
