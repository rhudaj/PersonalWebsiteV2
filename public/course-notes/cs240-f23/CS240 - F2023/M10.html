Huffman’s (encoding) Algorithm

How to build the best trie (shortest CCC﻿)

Idea

Infrequent characters should have longer codewords (further down the trie)

Pair up most infrequent characters.

Requires each nodes to have a weight; sum of frequencies in subtree

→ (Corresponds to adding one bit to the encoding of each character)

Algorithm

  1. Start with a set of encoding-tries, 1 for each c∈Σsc\in\Sigma_sc∈Σs​﻿

    * Stored in a min-oriented priority queue

  1. Merge the 2 tries with the min-weight

    * delete-min, delete-min, then insert

  1. Repeat step 2 until 1 trie left

  • Code

    Pros:

    • The constructed Trie results in CCC﻿ being short as possible

    Cons

    1. the trie is not unique (unless we give tie-breaking rules)
    1. Decoder does not know the Trie
      • we have to convert the trie first to a string, and then send that string
      • Or we can send character frequencies & tie-breaking rules along
      • Either way, this adds to the length of the encoded text
    1. encoding requires passing through the text twice (once to compute frequencies and another to encode). Cannot use a stream twice unless it can be reset

    Run-Time

    O(∣ΣS∣log∣ΣS∣+∣C∣)O(|\Sigma_S|log|\Sigma_S|+|C|)O(∣ΣS​∣log∣ΣS​∣+∣C∣)﻿

    • building encoding trie (delete-min or insert) → log(size of priority queue) = log(∣ΣS∣)log(|\Sigma_S|)log(∣ΣS​∣)﻿
    • The priority queue starts at ∣ΣS∣|\Sigma_S|∣ΣS​∣﻿ and reduce size by 111﻿ each time
  • Example

    S=GREENENERGYS=GREENENERGYS=GREENENERGY﻿

    ΣS={G,R,E,N,Y}\Sigma_S=\{G,R,E,N,Y\}ΣS​={G,R,E,N,Y}﻿

Run-Length Encoding
  • Requires: SSS﻿ and CCC﻿ both bitstrings
  • multi-character encoding: multiple characters can have 1 code-word
  • Decoding dictionary is uniquely defined nor explicitly stored

Idea

Some substrings are much more frequent

  1. Find these substrings
  1. Adaptive Encoding (start with a ASCII dictionary, update it)

    Encoder and decoder must both know how the dictionary changes

Encoding

  • Idea

    Build a Dictionary where the keys are words → Trie

    Convert SSS﻿ to list of ASCII codes, then:

    Every step adds to dictionary a multi-character string, using code-numbers 128, 129, … :

    1. Find longest string that is already in the Dictionary, encode the whole word.

    1. Add the substring that would have been useful to the Dictionary (add the word + the character after)

Burrows-Wheeler Transform
  • Encoding

    Assumes:

    • SSS﻿ is in an array (not a stream)
    • SSS﻿ ends with $\$$﻿

    S = alf␣eats␣alfalfa$

    1. Write shifts
    1. Sort cyclic shifts
    1. Extract last characters from sorted shifts

      Result:

      • Substring alf occurs three times
      • causes runs lll and aaa

    Fast way:

    • AsA^sAs﻿ is the sorting permutation of cyclic shifts = suffix array (sorted suffixes)
      • computing suffix array takes O(nlogn)O(nlogn)O(nlogn)﻿ time
    • Given just the permutation, we can identify the BWT encoding
  • Decoding

    Given CCC﻿, reconstruct the first and last column of the matrix of cyclic shifts by sorting

    C=ard$rcaaaabbC=ard\$rcaaaabbC=ard$rcaaaabb﻿

    1. Last column: CCC﻿
    1. First column: CCC﻿ sorted
    1. Disambiguate by row index
    • Repeated characters are in the same order in the first and last column (the sort was stable)
      • ie: the 5’s in the first column are in the same order as the a’s in the last column
    1. Find SSS﻿
    • Find $\$$﻿ in end column
    • Look at corresponding 1st column: (c,i)
      • add c to SSS﻿
      • look at the i-th row on the end column
    • Starting from $\$$﻿ (in the last column), recover SSS﻿ (character in the 1st column)

      S=abracadabra$S=abracadabra\$S=abracadabra$﻿
