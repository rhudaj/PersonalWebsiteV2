Protection

Pre-Emption

  • Give/take away resources from apps as needed

Mediation

  • Place OS between the App & Hardware
  • Track parts that the App is allowed to use (table)

Privilege Modes

  • Apps are unprivileged → user-space
  • OS is privileged → kernel-space
  • Protection operations can only be done in privileged mode
    • e.g, seg-fault for bad use of pointers

Typical OS Structure

Kernel

  • Inside the OS.
  • A program (w’ highest privilege)
  • Manages: any interaction with the hardware, memory, IO devices

Software usually runs in unprivileged mode

Kernel runs in privileged mode

System Calls

overview

system calls

  • let’s user run code specified by the kernel (which has privilege)
  • so apps can do low-level things

Steps

  1. The user calls a syscall in user-space
  1. Control is transferred to the kernel
  1. The kernel run’s that syscall in kernel-space
  1. Control is transferred back to the app

User Libraries

Library functions are wrapper’s for syscall’s

  • they live and run in the user-space
  • e.g, printf: calls write in kernel
