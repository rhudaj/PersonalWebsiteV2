Updates

INSERT

Insert Tuple(s)

  1. single tuple
  1. multiple tuples from query result
  • Syntax

    Single Tuple

    INSERT INTO T VALUES (c1, ..., ck)

    Adds tuples (c1, ..., ck) to table T

    Multiple Tuples

    INSERT INTO T VALUES (Q)

    Adds each tuple computer by Q to table T

  • EX
    INSERT INTO author (
	SELECT max(aid) + 1, 'Tim'
	FROM author
)

DELETE

Remove all tuples

  1. satisfying a condition
  1. using cursors (later)
  • Syntax

    Using a Condition

    DELETE FROM T WHERE <condition>

    Deletes all tuples that match <condition>

  • EX

    Delete authors who have not written anything

    DELETE FROM author 
WHERE NOT EXISTS (
	SELECT * FROM WROTE
	WHERE author = aid
)

Assertion

Creates a general integrity constraint

  • Syntax
    CREATE ASSERTION <assertion-name>
CHECK (<condition>)
    • identifier: <assertion-name>
    • Transaction fails if it were to make <condition> false
  • Examples
    • Ex: primary key

      No pairs of publications have the same pubid

      CREATE ASSERTION unique-pubid
CHECK (
	NOT EXISTS (
		SELECT * 
		FROM publication p1, publication p2
		WHERE p1.pubid = p2.pubid
		AND p1.title != p2.title 
	)
)

      In RC

      ∀p,t1,t2:PUBLICATION(p,t1)∨PUBLICATION(p,t2)→t1=t2\small{ \forall p, t_1, t_2:PUBLICATION(p, t_1) \lor PUBLICATION(p, t_2) \rarr t_1 = t_2 }∀p,t1​,t2​:PUBLICATION(p,t1​)∨PUBLICATION(p,t2​)→t1​=t2​﻿

    • Ex: Foreign Key

      Every author in a wrote tuple occurs as an aid in the author table 

      CREATE ASSERTION author-foreign-key
CHECK (
	NOT EXISTS (
		SELECT * 
		FROM WROTE w
		WHERE NOT EXISTS (
			SELECT * 
			FROM author a
			WHERE a.aid = w.author
		)
	)
)

      In RC

      ∀a:WROTE(a,−)→AUTHOR(a,−)\small{ \forall a: WROTE(a,-) \rarr AUTHOR(a,-) }∀a:WROTE(a,−)→AUTHOR(a,−)﻿

Views

Like a table:

  • appears in schema
  • Can be Queried
  • Other views can be defined in terms of it
  • Syntax
    CREATE VIEW <view-name> AS (<query>)
  • EX

    A view called PUBSWITHAUTHORS with ids, titles, author count of publications with at least 1 author:

    CREATE VIEW pubswithauthors AS (
	SELECT DISTINCT pubid, title, count(*) AS acnt
	FROM publication, wrote
	WHERE pubid = publication
	GROUP BY pubid, title
)

    Using the view in a query: 

    SELECT * FROM pubswithauthors

Triggers

  • Syntax: 
    CREATE TRIGGER [name]
AFTER [event] ON [tableOrView]
	[ REFERENCING OLD AS [corelation-old] ]
	[ REFERENCING NEW AS [corelation-new] ]
	FOR EACH ROW [ WHEN [condition] ] BEGIN ATOMIC 
		[dmlAction_1];
		[dmlAction_2];
		...
END
  • EX

    When deleting an author, also delete wrote tuples.

    • event - DELETE ON author
    • action - DELETE from wrote
    CREATE TRIGGER delete_author
AFTER DELETE ON author
REFERENCING OLD AS a
FOR EACH ROW BEGIN ATOMIC
	delete from wrote where wrote.author = a.aid; 
END

Timing of integrity constraint checking must be managed carefully

→ deferring all COMMIT time always works

  • Rules with Foreign Keys

    Special syntax for common rules related to foreign key constraints: 

    FOREIGN KEY (Some Attributes) REFERENCES (Some Attributes) ]
		ON DELETE <action> 
		ON UPDATE <action>

    Where <action> can be:

    • RESTRICT (produce an error)
    • CASCADE (propagate the delete)
    • SET NULL (set to unknown)
  • EX
    CREATE TABLE WROTE (
	author integer not null,
	publication integer not null,
	primary key (author, publication),
	foreign key (author) references AUTHOR 
		ON DELETE cascade, 
	foreign key (publication) references PUBLICATION
)
