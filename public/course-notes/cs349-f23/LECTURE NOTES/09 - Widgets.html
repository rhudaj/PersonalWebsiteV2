Characterizing Widgets

Ways to characterize:

  1. Essential Function

    What it represents (not how it looks)

  1. Container/Not
  1. Behavior
  1. Appearance

Logical Input Device

Widgets are specific instances of Logical Input Devices

Examples

  • Logical Button Device

    Essential Function:

    1. Take 1 input
    1. Generates “action” events

    Instances:

    • Button Widget
    • Drop Down Item
    • keyboard shortcut

  • Logical Number Device

    Essential Function:

    1. Display Value
    1. Send “value changed” event

    Optionally

    • accept number as input
      • validation (ie: range)

    Instances:

  • Logical Boolean Device

    Essential Function

    1. Display & Input Boolean Value
    1. Sends “value changed” event

    Instances

  • Logical Text Device

    Essential Function:

    1. Display &/or Input Text Value
    1. Sends “value changed” event

    Instances:

  • Logical Choice Device

    Essential Function:

    1. Input & Display 1 choice among list
    1. Sends “selection changed” event

    Instances:

    • group of radio buttons

Widget Toolkit Goals - Reusability

  1. Complete - all you need
  1. Consistent - behavior across component
  1. Customizable

Customization

  • change some appearance and behavior
    • ie: colors, font, orientation
  • Factor out key behavior
    • formatters
    • LayoutManagers
  • Expose Class Hierarchy that promotes customization
    • ie: Button extends ButtonBase (not just Button extend SKElement), base class implement a logical button device
