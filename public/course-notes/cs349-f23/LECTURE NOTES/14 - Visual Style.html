Colour

Colour Terminology

  • Hue and saturation
  • Tint, tone, shade

Colour Theory: Harmonious Palettes

  • different ways to get base sets of hues
  • then you change the saturation and luminance

Colour Perception

  • Human ability to discriminate colours depends on context
  • It’s harder to tell two colours apart when:
    • the colours are pale (A)
    • the object is small or thin (B)
    • the colour patches are far apart (C)

Colour Hierarchy and Semantics

  • A colour design system defines how colours are used, typically two sets:

Primary Colours

These are not primary and secondary colours from colour theory

  • used frequently in screen/page designs to create experience,
  • includes tints, shades, and tones of primary brand colour(s) (e.g. black, grey, white or similar subtle low-saturation colours)
  • colours for branding, interaction elements, layouts, text

Secondary Colours

  • used occasionally in designs or in interactive elements
  • used for functions: task success, error, warning; selected element; etc.
  • colours are accents based on primary brand colours

Colour Rules of Thumb

Keep palette simple

  • consider one primary brand colour with monochromatic palette with grey tones as a start
  • max 3 colours for primary palette( you can always generate tins, shades, and tones as needed)
  • Use secondary colours to support usability and functionality
    • be consistent and establish a colour language for interaction

Typography

“Typeface” and “Font”

  • Typeface Anatomy
    • Baseline, bottom of std lowercase letters
    • Ascender — part of text above baseline + std lowercase height
    • Descender — distance below baseline for std lowercase height
    • Leading — (line height) Distance between baseline and baseline between lines
    • Kerning — overlapping width between letters
  • 4 Main Classes of Typefaces
    • serif
    • sans serif
    • display
    • script
  • typeface’s have fonts (common misconception)
    • Times, Helvetica, Arial are typefaces
    • Times 12pt Bold is a font
  • Font Anatomy
    • Weight (e.g. bold)
    • Emphasis (e.g. italic)
    • Point size

Choosing Typefaces

  • Prefer standard, simple typefaces
    • Sans-serif are most readable and work best on digital platforms 
    • Serif can be effective for headings and feels classic and stable
  • In general, avoid display and script typefaces
  • ie: Comic Sans (is sans-serif)
    • Comic Sans is weighted in a way that looks uneven (Look at the letter n).
    • It skews characters, which makes them look unbalanced
    • kerning is inconsistent

Typographic System

  • Define hierarchical names 
    • e.g. body, caption, heading, title, …
  • Use size, weight, and emphasis to create visual hierarchy
    • 7 different scales is often enough: e.g. 14, 16, 18, 24, 32, 48, 72
  • Use all caps sparingly for headings

Leading and Paragraph Spacing

  • Leading is space between lines
  • Increase leading to emphasize lines in paragraphs
    • try 140% to 180%
  • Add space between paragraphs rather than indent first line

Alignment

  • Use left alignment for text in paragraphs
  • Use left or centre alignment for text in headings
  • Use centre alignment for buttons and similar interface controls
  • Never use fully justified text
    • harder to read, creates unintentional "rivers" which looks bad

  • linking to Google Web Font
  • Defining a modular font system in CSS 
    • using ems as font size
    • CSS var and calc
    • inheritance for heading styles
    • font-weight
    • line-height
    • "lorem ipsum" placeholder text

Rules of Thumb

  • Avoid using display typefaces (especially avoid comic sans)
  • Avoid using more than 2 typefaces in a design (1 is often best)
  • Avoid using more than 3 weights (avoid Black and Ultralight weights)
  • Avoid underlining unless it's a hyperlink (use bold and italics for emphasis)
  • Avoid fully justified text

Layout

  • The Grid
    • Underlying regularized structure to organize elements in a layout.
    • Most basic form is vertical columns and gaps (aka "gutters").
      • often 12 columns (because 12 can be divided by 2, 3, 4, ... ) 
    • Often includes a "baseline" grid to place text

    8-Point "Pixel Grid"

    • Use multiples of 8 to define dimensions, padding, and margins 
      • column and gaps are multiples of 8
      • element sizes and spacing are multiples of 8
      • even type can be multiples of 8 (though often multiples of 4)
  • Negative Space

    "white space"

    • It groups and divides content
    • It doesn't need to white or even one colour, can be a gradient, background image, texture(but must not contain content)

  • Alignment
    • alignment must be done on visual parts of an element, which may not be the same as underlying file or bounding box
    • Using a grid forces good alignment
  • Aspect Ratio
    • Don't EVER change the aspect ratio of an image, illustration, or type to fit a layout.

Rules of Thumb

  • Use an underlying grid to guide your design - you can create grid layouts with flexbox
  • Don't try to use all of the power of the grida 12-column grid doesn't mean use all permutations of columns- keep it simple
  • It's ok to "break" the grid for added emphasis
  • Use negative space, don't make your layout too dense
