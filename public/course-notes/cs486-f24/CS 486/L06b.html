Decision Trees
  • nodes: input attributes/features
  • branches: input features values
    • can be labelled with multiple feature values
  • leaves: predictions for target features (point estimates)
(Generic) Learning D.T Algorithm

Idea: recursion

  • for non-binary features, you may want to group features to limit the # children.
  • Up to us: stopping_criterion, selectFeature & pointEstimate
def DecisionTreeLearner(
    X: list, # input features
    Y: list, # output features
    E: list, # set of training examples
):
    # BASE CASE
    if stopping_criterion(X, Y):
        return pointEstimate(Y)

    # RECURSIVE CASE
    Xi = selectFeature(X, Y)
    # Split E into subsets Ei based on Xi, recursive call on each.
    subTrees = [
	    DecisionTreeLearner(
		    X - Xi,
		    Y,
		    filter(lambda e_i: e_i[Xi] == Xi_val, E)
		  )
	    for Xi_val in Xi
    ]

    return (Xi, subTrees)     # decision tree (leaf || subtree)
  • stopping_criterion
    • Related to final return value
    • Depends on our goal.
    • Possible criteria:
      1. No more features
      1. Performance on train-set “good enough”
  • selectFeature
    • Ideal: choose features (order) that result in smallest (simple) tree (hard as the problem itself)
    • Reality: myopically split: as if only allowed 1 split. Which feature would give “best” performance.
    • “Best” performance, options:
      1. Most even split (2 peices)
      1. Max info gain (from doing split)

        …

    • Example of “good” vs. “bad”
  • pointEstimate
    • Point estimate of Y (output features) over all examples
    • Point estimate is just a prediction of target features
    • Examples:
      • mean value
      • median value
      • most likely classification

Learning DT Alg w’ Priority Queue

The previously learning algorithm was recursive; recurse on each feature.

We can improve our feature selection by prioritizing highest-info-gain features:

  • Sort leaves using a P.Q ranked by info-gain, with the best feature at that leaf.
  • Always expand the leaf at the top of the queue.

def DecisionTreeLearner(
    X: list,  # set of input features (column names)
    Y: list,  # target feature (labels for each row)
    E: list,  # set of training examples (rows)
):
	''' Output: Decision Tree '''
		DT = pointEstimate(Y,E) # initial decision tree.
		X', IG = selectFeature(X,E)
		PQ = {DT, E, X', IG} # priority queue of leaves ranked by info-gain

		while not stoppingCriteria(...):
				{Sl, El, Xl, IGl} = PQ.head()    # leaf

						Ei = e in E if E[Xl] = v_i
						Xi, IGi = selectFeature(X \ {Xl}, Ei)
						Ti = pointEstimate(Y,Ei)               # new subtree
						PQ.add({Ti, Ei, Xi, IGi})
						Sl.addChild(Ti)
		return DT
Overfitting

Overfit: the DT is too good at classifying training data and does not generlize well.

  • Often occurs when there is not enough data

Methods to avoid

  1. Regularization: stop training early (when DT is big enough)
  1. Pseudocounts: generate fake data (noise) based on seen data.
  1. Cross Validation: split off some of the train-set, remove labels, use as test-set.

Test Set Errors Caused by

  • Bias: error due to alg. finding imperfect model
    • representation bias: model too simple
    • search bias: not enough search
  • Variance: due to lack of data
  • Noise: data depends on features not modeled
  • Bias Variance Tradeoff:
    model___ data__ bias__ variance
    complicatednot enoughlowhigh
    simplelots of highlow
Capacity
  • inverse of bias
  • ability of model to fit variety of functions
  • A D.T model has low capacity, high bias
