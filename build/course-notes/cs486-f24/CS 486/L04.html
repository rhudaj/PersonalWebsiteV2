2. Backtracking

Assign variables one at a time:

  1. Ordering: decide which order you will assign values to the variables.
  • Problem: Finding the optimal ordering is as hard as the problem.
  • Solution: Push failures as high as possible. Cut off large branches of the tree as soon as possible.
  1. Pruning: After each variable assignment, check constraints (even before assigning values to the remaining variables)
3. Consistency
  • goal: each domain takes a value, and all constraints are satisfied.
  • Backtracking would “re-discover” this inconsistency.

Consistency Network (CN)

  • circles = variable domains
  • squares:
    1. domain constraint: c(X)
      • circle → box
    1. arc constraint: C(X,Y)
      • circle → box → circle

Domain Consistent X: X satisfies all domain constraints

Domain Consistent CN: all nodes are domain consistent

Arc Consistent Arc: for any X value, there is a Y value that satisfies c(X,Y)

Arc Consistent CN: every arc is arc consistent

Path Consistency:

X1,X2,X3,...,XN{X_1, X_2, X_3, ..., X_N}X1​,X2​,X3​,...,XN​﻿ is path consistent if:

  1. domain consistent CN
  1. arc consistent CN
  1. For any triplet of variables (Xi, Xj, Xk), if values for Xi and Xj satisfy the constraint between them, there must be some value for Xk that is consistent with both Xi and Xj.

AC-3 Algorithm

Domain consistency + Arc Consistency 

def AC3(csp: CSP):
	# 1 -- Domain Consistency
    csp.makeDomainConsistent()

	# 2 -- Arc Consistency
    TDA = list(csp.binaryConstraints)  # TDA: To Do Arcs queue

    # For each arc constraint, revise the domain of X
    while len(TDA) > 0:

        bc: Constraint = TDA.pop(0)
        X, Y = bc.vars
        Dx, Dy = csp.domains[X], csp.domains[Y]

        # Revise the domain of X
        revised = False
        # Remove x values where no y value satisfies the constraint
        for x in Dx:
            if not any(bc.check([x, y]) for y in Dy):
                Dx.remove(x)
                revised = True

        if revised:
            if not Dx:
                return False    # NO SOLUTION

            otherBCs = filter(
                lambda bc: ( bc not in TDA and bc.vars[0] != Y and bc.vars[1] == X ),
                csp.binaryConstraints
            )

            # TDA = otherBCS + TDA
            TDA = list(otherBCs) + TDA

    return True # ARC CONSISTENT
  • Guaranteed to terminate

    When AC-3 terminates, either:

    1. Every domain is empty ⇒ no solution, OR
    1. Every domain has a 1 value ⇒ solution, OR
    1. Some domain has more than one value ⇒
      • Split it in two
      • run AC-3 recursively on two halves
        • Don’t have to start from scratch — put back all arcs ⟨Z, c′(Z, X)⟩ if X was the domain that was split
  • Complexity — O(cd3)O(cd^3)O(cd3)﻿
    • nnn﻿ variables, c binary constraints, size of each domain is at most d
    • Each arc (Xk, Xi) can be added to the queue at most d times because we can delete at most d values from Xi
    • checking consistency of each arc can be done in O(d2) time

5. Local Search

Idea: (need to define)

  • Maintain an assignment of a value to each variable.
  • At each step, select (based on heuristics) a neighbour of the current assignment.
  • Stop when a satisfying assignment is found (or best found)

For CSP

Heuristic: # conflicts

⇒ select by minimizing the heuristic

⇒ goal: assignment with 0 conflicts (unsatisfied constraints) constraints

Greedy Descent

  1. A plateau where the heuristic values are uninformative

Randomized Greedy Descent

on top of downward steps, can have:

  1. Random steps: move to a random neighbour, OR
  1. Random restart: reassign random values to all variables

(1) + (2) = stochastic local search

Tabu Lists

Idea:

  • GSAT (Greedy SATisfiability): never choose same variable twice
  • maintain a tabu list of the k last assignments (to prevent cycling)
  • Don’t allow an assignment that is already on the tabu list

Parallel Search

A total assignment is called an individual

  • maintain a population of kk k﻿ individuals instead of one
  • At every stage, update each individual in the population
  • report individuals who are solutions
  • Like k-restarts, but uses kkk﻿ times the min-number of steps

Beam Search

  • parallel search, but choose k best out of all of the neighbours
  • When k = 1, it is greedy descent
  • The value of k lets us limit space and parallelism

Stochastic Beam Search

  • Like beam search, but it probabilistically chooses the k individuals at the next generation.
  • The probability that a neighbor is chosen is proportional to its heuristic value : e−h(n)/Te^{-h(n)/T}e−h(n)/T﻿
  • This maintains diversity amongst the individuals
  • The heuristic value reflects the fitness of the individual
  • each individual mutates and the fittest ones survive

Genetic Algorithms

  • Like stochastic beam search, but pairs of individuals are combined to create the offspring
  • For each generation (Stop when a solution is found)
    • Randomly choose pairs of individuals where the fittest individuals are more likely to be chosen
    • For each pair, perform a cross-over: form two offspring each taking different parts of their parents
    • Mutate some values
