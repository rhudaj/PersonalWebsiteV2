BST

  • search(k)
    1. start at root
    1. compare k to current key
      1. Go left or right
      1. STOP if:
        1. subtree empty
        1. key found
  • insert(k, v)
    1. search(k)
    1. insert (k,v) where you end up
  • delete(k)
    1. search(k)
    1. Look at where k is:
      1. leaf → delete it
      1. 1 subtree → move up
      1. 2 subtrees →
        1. x ← successor
          • the minimum in the right subtree
          • go left as far as you can in k.right
        1. swap k & x
        1. delete(x)

Height of a BST

    If n items are inserted, how big is h?

    • Average Case

      h∈Θ(logn)h \in \Theta(logn)h∈Θ(logn)﻿

      * if keys are inserted in random order

AVL Tree

BST + Height Balance Property

  • heights of any L/R subtree differ by at most 1
    • balance(v)=height(R)−height(L)\blue{balance(\red{v})}=height(\red{R})-height(\red{L})balance(v)=height(R)−height(L)﻿
      • -1 → left heavy
      • 0 → balanced
      • 1 → right heavy
  • Example

  • AVL Tree with nnn﻿ nodes has Θ(log⁡n)\Theta(\log{n})Θ(logn)﻿ height
    1. Find formula for N(h) = smallest h for n nodes

    1. How does N(h) grow?
    1. We can get a upper bound for h

      Given any AVL tree with height = h, # nodes = n:

      n≥N(h)≥2h=2h/2n \ge N(h) \ge \sqrt{2}^h = 2^{h/2}n≥N(h)≥2​h=2h/2﻿

      log(n)≥log(2h/2)log(n) \ge log(2^{h/2})log(n)≥log(2h/2)﻿

      log(n)≥h2log(n) \ge \frac{h}{2}log(n)≥2h​﻿

      h≤2log(n)h \le 2log(n) h≤2log(n)﻿

    1. We already know the lower bound for h

      Given any AVL tree with # nodes = n:

      h is minimal when the each level is filled to the max:

      n≤2h−1+1n \le {2^{h-1}}+1n≤2h−1+1﻿

      …

      log(n−1)+1≤hlog(n-1)+1 \le hlog(n−1)+1≤h﻿

    1. Summary:

      log(n−1)+1≤h≤2log(n)log(n-1)+1 \le h \le 2log(n) log(n−1)+1≤h≤2log(n)﻿

      h∈Θ(logn)h \in \Theta(logn)h∈Θ(logn)﻿

Fixing Unbalanced Subtree

Goal: fix structure without altering order of keys

  • Many different BSTs with the same key order

    In all pictures → same inorder traversal

    • A - 20 - B - 30 - C - 40 - D

  • restructure(z)

    Where z is the unbalanced node

    Step 1: get c,gc,gc,g﻿

    • for ggg﻿, break ties to prefer single rotation

    Step 2: use order of g,c,zg,c,zg,c,z﻿ to identify rotation

    Step 2: place children of g,c,zg,c,zg,c,z﻿ will be in the same order they were before
