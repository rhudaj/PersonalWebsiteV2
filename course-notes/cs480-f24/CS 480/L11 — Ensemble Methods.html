Topic: Ensemble Methods

A combined model can be more powerful than any of the individual models.

Recall the Bias-Variance Tradeoff

  • High Variance — Models would be quite different across different training sets (boundary depends too much on each decision point)
  • High Bias — All models would be quite similar.

Recall: SVMs

  • CCC﻿ controlled the tradeoff
    • large ⇒ any point greatly effect the decision boundary
  • In Linear Regression, we used R.R parametrized by λ\lambdaλ﻿, to control this tradeoff (penalized norm of the weights).

Bias/Variance related by Test Error

Expected error for a new test point (test error) is the expected error across a bunch of models for that test point:

  1. Irreducible Error: inherent random noise of true fff﻿
  1. Squared Bias: difference between expected value of our predictions and true fff﻿
  1. Variance: the difference between a particular model’s prediction and the average prediction made by the models.

So far we’ve been searching for a sweet-spot that minimizes test-error (the dip in red), tolerating a bit of both (e.g. with cross validation)

Characterizing models:

  • High bias & high variance: (worst case), under-fit while also sensitive to changes in the data. Can happen if you have poor data, or your not using the right hypothesis class.
  • Low Bias, Low Variance (ideal): we want near perfect predictions always. C
    • Can’t achieve this with a single hyper-parameter; have to tolerate some bias/variance. We’re currently stuck trading them off, finding a balance.

In this course we’ve mainly been fitting a single model to a batch of data, and using that for all future predictions.

Ensemble Methods

Combine multiple methods to improve performance.

Parallel vs Sequential

2 Methods: parallel, sequential.

Parallel

  • Train a model for different version of the same batch of data
  • Variance of the ensemble < variance of individual model

Sequential

Given batch of data, train the first (base) model on it, create the another one based on the performance of the last one. Each fills in what the previous could not handle.

(tendency) The Bias of the ensemble is less than any individual model.

Reducing Variance

Idea:

Given more data, by combining multiple models, we can achieve more stable results (lower variance) with the same bias by partitioning the data, training a model on each set, and then aggregating the results.
  • Example

Bootstrapping

  • what?
    • Reduce variance for a fixed dataset
  • How?
    • Resample with replacement.
      • same # samples, but each sample can appear any # of times
    • Each bootstrapped sample has a slightly difference mix of points.
  • Why?
    • each model has a slightly varying data ⇒ prevents overfitting
  • Assumption
    • the original population is a good estimate of the population.

      ⇒ the average of all bootstrapped samples (for large amounts) will approach the dataset average, and hence, the true average.

  • Example 1

    Say we have a sample S1S_1S1​﻿ from some population. We find it’s mean to be 1.01.01.0﻿.

    If we double our data, we can now get the mean of 2 groups. We find the mean of S2S_2S2​﻿ to be 0.80.80.8﻿. So our new estimate of the population mean will be 0.90.90.9﻿.

    Notice that using samples from S1S_1S1​﻿, we derive the mean of S2S_2S2​﻿.

Random Forests

  • De-correlates samples (like before) and features.
  • selects a random subset of mmm﻿ predictors.
  • So each tree is not influenced by the same features. Each model gets a different idea of what features to make decisions on.
  • Why? If one feature dominates, all trees will use that for the root node, resulting in very similar trees. When the trees are correlated this way, averaging there predictions does not reduce the variation as well.
  • Random forests have a great middleground between bias and variance.

Reducing Bias (boosting)

Boosting in General

  • MMM﻿ base models, yiy_iyi​﻿. Each has a set of weights, wn(i)w_n^{(i)}wn(i)​﻿, one weight for each point in the dataset. The weights used to train the ithi^{th}ith﻿ model are determined by the i−1thi-1^{th}i−1th﻿ model.
  • i.e. One model has control over the relative weighting of points used to fit the next model
  • Similar to bootstrapping (e.g. having points show up multiple times), but it’s controlled by the previous model.
  • We still aggregate over all models, but its a bit more complicated.

Purpose

  • Boosting is meant to combine weak models and turns them into a strong model.
    • Weak = biased (not much better than guessing)
  • In boosting, models depend on each other. In bagging, they needed to be independent.

ADA Boost

We increase the weights of points that were correctly classified and decrease the weight of those that weren’t. Essentially, the predictions from all previous models are added together as the ensemble grows. The next model tries to correct the mistakes of the previous ones by fitting the weighted errors. The final prediction, Ym(x)Y_m(x)Ym​(x)﻿ is a weighted sum of the models predictions, αi\alpha_iαi​﻿ for model iii﻿.

Algorithm

  • final prediction = weighted sum of individual models. predictions.
    • Each classifier has parameters that are learned based on the weighted misclassifications
    • A weighted error rate is calculated to determine how well the current model performs with the weights
    • The importance of the current classifier is calculated such that lower errors result in higher weights, which has more influence in the final decision
    • The weights of the data points are updated such that misclassified points get more weight
    • The weights for each data point are adjusted based on the performance of the previous model such that data points that are misclassified are given more weight
