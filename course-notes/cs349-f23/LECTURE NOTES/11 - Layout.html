Box Model

Hierarchy of 4 dimensions:

  • margin
  • border
  • padding
  • content

How to define box (w,h)

  • Standard CSS
    • (w,h) ‚Üí content size
    • render size ‚Üí content size + padding + border
  • Alternative CSS
    • (w,h) ‚Üí rendered size
    • content size = rendered-sized - padding - border

SimpleKit - Box Model

Uses: CSS Alternative box model

3 Dimensions:

  • margin
  • padding
  • content

  • boxmodel class: 
    type Size = { width: number; height: number };

export class BoxModel {

	// each has a get() and set()
	private _margin = 0;
	private _padding = 0;
	private _contentWidth = 0;
	private _contentHeight = 0;

  onUpdate() {
    invalidateLayout();
  }

  // margin
  set margin(m: number) {
	  ...
    this.onUpdate();
  }

  get marginBox(): Size {
    return {
      width: this.contentWidth + 2 * this.padding + 2 * this.margin,
      height: this.contentHeight + 2 * this.padding + 2 * this.margin,
    };
  }

  set padding(p: number) {
	  this.padding = p;
		//adjust width and height to account for padding
		this.width = this.width;
		this.height = this.height;
    this.onUpdate();
  }

  get paddingBox(): Size {
    return {
      width: this.contentWidth + 2 * this.padding,
      height: this.contentHeight + 2 * this.padding,
    };
  }

  // content
  set contentWidth(w: number) {
    ...
    this.onUpdate();
  }

  private _contentHeight = 0;
  set contentHeight(h: number) {
    ...
    this.onUpdate();
  }

  get contentBox(): Size {
    return { width: this.contentWidth, height: this.contentHeight };
  }

  // full box size (including margin)
  get fullWidth() {
    return this.marginBox.width;
  }

  set fullWidth(w: number) {
    ...
    this.contentWidth = w - 2 * this.padding - 2 * this.margin;
  }

  get fullHeight() {
    return this.marginBox.height;
  }

  set fullHeight(h: number) {
    h = Math.max(0, h);
    this.contentHeight = h - 2 * this.padding - 2 * this.margin;
  }

  // width and height include padding

  // width
  set width(w: number) {
    // width must be at least 2 * padding
    w = Math.max(2 * this.padding, w);
    this.contentWidth = w - 2 * this.padding;
  }

  get width() {
    return this.contentWidth + 2 * this.padding;
  }

  // height
  set height(h: number) {
    // height must be at least 2 * padding
    h = Math.max(2 * this.padding, h);
    this.contentHeight = h - 2 * this.padding;
  }

  get height() {
    return this.contentHeight + 2 * this.padding;
  }

  // draw box model for debugging
  draw(gc: CanvasRenderingContext2D) {
    gc.save();
    // margin
    // padding
    // content
    gc.restore();
  }
}

Layouts

SimpleKit

Containers have a LayoutMethod:

type LayoutMethod = (
	boundsWidth: number,
	boundsHeight: number,
	elements: SKElement[]
) => Size; 
protected _layoutMethod: LayoutMethod | undefined;
set layoutMethod(lm: LayoutMethod) {
	this._layoutMethod = lm;
}

To assign layouts, you do: 

container.layoutMethod = layoutMethod({params})

You set the layout method of the container to a reference of a function.

We can modify the function that we pass with the {params} part

FixedLayout

  • simplest layout type
  • does not adjust to canvas resize, does not adjust w/h, x/y of elements

Center Layout

  • all elements centred in container, stacked back to front
  • sets new x and y for each element
  • still does not adjust element height/width

Wrap Layout

  • places elements in rows, wrapping to next row as needed
  • takes a gap parameter, when making the function
root.layoutMethod = Layout.makeWrapRowLayout({ gap: 10});

Responsive vs Adaptive

Both layouts that can change (resize)

ResponsiveAdaptive
Support a universal design that reflows spatial layout to fit the dimensions of the current view (device or window)Design optimized spatial layouts for each device and dynamically switch to fit devices

Widget ‚ÄúPreferred‚Äù Size

  • widgets should be flexible in size and position
  • the layout can change a widgets size and position
  • the widget should be able to set some bounds for what‚Äôs allowed.

SimpleKit:

  1. widget specify (w,h) or,
  1. undefined ‚Üí widget lets the layout choose one

Variable Intrinsic Layouts

  • layouts that do this responsive/dynamic sized layout
  • layout attempts to use the widget preferred sizes, but queries all widgets first and allocates space to them as a group

Layout is determined in 2 passes:

  1. get each child widget‚Äôs preferred size
  1. Decide on a layout that satisfies everyone‚Äôs preferences. Then iterate through each child and set its size/position.

fill layout

  • implements dynamic layout
  • lays out elements in a row, they can grow or shrink to fit space.
  • takes a gap parameter

Layout Invalidation

To avoid changing the layout every time you change a parameter

  • every time widget size/layout related property changes, at least some of the UI has to be laid out again
  • there could be multiple changes each frame of the run-loop
  • Best-Practice: run layout at most once per run-loop frame
    • SimpleKit uses a global function and flag (in run-loop.ts) ‚Üí InvalidateLayout(), layoutDirty
    • Whenever the widget size or layout related property changes, it calls InvalidateLayout() but the layout isn‚Äôt done until the start of the next frame of run-loop.
    if(uiTreeRoot && layoutDirty) {
	layoutRoot()
	layoutDirty = false;
}
    • layoutRoot() ‚Üí sets root to canvas size and calls doLayout()which will call doLayout for all widgets in the widget tree
