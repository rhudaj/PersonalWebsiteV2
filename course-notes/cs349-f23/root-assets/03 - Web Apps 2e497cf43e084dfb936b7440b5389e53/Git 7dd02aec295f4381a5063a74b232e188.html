Git

  • Working Directory:
    • local copy
  • Staging Area:
    • intermediate area where commits are reviewed before completing the commit
  • Repository:
    • a data structure of commits, usually there's a "local repo" on your machine and a "remote repo" on a server

Commands

initcreate new empty local repo
addadd file from working directory to staging area
commit• commit changes from staging area to local repo • snapshots of "tracked" files as they change over time–m "description" - give it a description
statusDisplay list of changed files and staged files
pushsend commits in local repo to remote repo
pull merge commits in remote repo with local repo
clonemake a copy of a repoarg: username@host:/path/to/repository
stashIf you want to preserve your local changes, you can safely store them on a Stash. They will be available in case you want them back at a later point.--include-untracked
resetdiscard your local changes completely:--hard

Typical Workflow

1git clone username@host:/path/to/repositoryGet copy of a remote repository on your computer
2git add <file> git rm <file> git move <old name> <new name>Update your source code in the local repo • new file • delete file • rename file
3git add -AAdd all source-code updates to staging\
4git statusCheck the staging status to verify everything is ready to commit
5git commit –m "desc of this commit"commit
6git pushpush changes to remote repo
7Go to step 2

.gitignore

  • file to specify intentionally untracked files that Git should ignore
    • ie: ignore windows, mac or linux stuff
  • Good practice to create a master gitignore for a repo in the root
