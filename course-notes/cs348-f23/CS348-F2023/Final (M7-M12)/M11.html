Logging

Approach to recovery

Every time a transaction updates an object, an UNDO/REDO record is created

COMMIT (DURABILITY)

  • guarantee permanent changes
  • redo changes of a committed transaction

ABORT (ATOMICITY)

  • guarantee no changes
  • perform all UNDO's for the transaction

Failure

  • GO UP, UNDO aborted
  • GO DOWN, REDO committed
