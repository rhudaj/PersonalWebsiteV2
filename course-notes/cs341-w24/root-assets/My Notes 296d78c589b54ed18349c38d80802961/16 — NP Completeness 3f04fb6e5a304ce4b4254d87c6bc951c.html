NP Completeness

Informally, a problem is NP-Complete if it is the hardest problem in NP

  • All problems in NP can be reduced to it
  • it’s the hardest (can be used to solve the other problems)

How to show a problem is NPC?

Option 1

Show that every NP problem reduces to it.

Option 2

  1. Show that it’s NP
  1. Show that another problem in NPC reduces to it

Known NPC Problems

  • 3-SAT, SAT
  • IS, VC, Clique
  • HC, HP
  • Subset Sum
    • Given a set of non-negative integers and a value sum, the task is to check if there is a subset of the given set whose sum is equal to the given sum. 
  • 0/1 Knapsack
    • Given N items where each item has some weight and profit associated with it and also given a bag with capacity W, Does there exist a subset of the items whose sum of weights is ≤W such that the sum of the associated values is ≥ V ?

  • instance: a DAG representing a circuit
    • inputs labeled with boolean variables: xix_ixi​﻿

Circuit-SAT is NPC

  • Idea

    Firstly, circuit-SAT is NP: the verification is just running a truth assigment through it and seeing if the output is yes or no. It runs in poly-time since the size of ttt﻿ is bounded by the size of SSS﻿

    Any problem in NP, can be reduced to an instance of circuit SAT.

    We can reduce the verification algorithm for the NP problem to a circuit.

    For any problem, AAA﻿, in NP

    This verifcation algorithm can be turned into a circuit.

    • ie: we want to know if you can you find a truth assigment for that circuit that turns the circuit true (that becomes your solution / certificate, ttt﻿)

    Then, the verification algorithm runs in poly-time.

    circuit-SAT is in NP & every NP problem reduces to it ⇒ NPC
