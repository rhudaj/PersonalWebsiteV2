Minimax (Search)

Used for Adversarial Search:

competitive, 2-person, zero-sum games

  • e.g. tic-tac-toe

Idea:

  • Min (player 1) & Max (player 2) nodes
  • Find the best option for player 1
  • Assume player 2 chooses worst-option
  • Recursively search to leaf nodes to find state evaluations, and percolate values upward through the tree.

Pseudocode: 

def minimax(node, depth, isMax):
    if depth == 0 or node.isTerminal:
        return H(node)
    if isMax:
        # you control this node
        bestValue = -float('inf')
        for child in node.children:
            V = minimax(child, depth-1, False)
            bestValue = max(bestValue, V)
    else:
        # you do not control this node
        bestValue = float('inf')
        for child in node.children:
            V = minimax(child, depth-1, True)
            bestValue = min(bestValue, V)
    return bestValue

Trade Optimality for Speed

Problem:

Can’t search to every leaf node for some games (e.g. chess)

Solution (Can no longer guarantee optimal play):

  • set a fixed maximum depth for the search tree, OR
  • stop at non-leaf nodes via heuristics

Bidirectional Search

  • Search from both directions, until you meet
  • Runtime: 2bk/22b^{k/2}2bk/2﻿
    • (maybe) exponential saving in time and space
  • Problem: making sure the frontiers meet
  • Example:
    • search 1: BFS to build set of locations that could lead to a goal
    • search 2: other search to find a path to these locations
